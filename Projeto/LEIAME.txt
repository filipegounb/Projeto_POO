Obs. 3: O projeto é enviado com alguns usuários já cadastrados. Para testar o sistema em seu estado inicial, apague
todos os arquivos da pasta 'dados' (recomendo que não delete o arquivo "Notas.txt" - ver Obs. 7).

Obs. 5: O código foi feito com o uso de system ("cls"), que limpa a tela. Se houver problemas na compilação, tente excluir todos os comandos
system ("cls") e compile novamente.

Obs. 6: Para executar os códigos direto do arquivo executável, faça: copie o arquivo executável de maneira q ele esteja no mesmo diretório do
arquivo do projeto. Ex.: para rodar o teste de unidade do mgd, copie o executável da pasta Projeto\Teste unitario MGD\TU_MGD\bin\Debug para a pasta Projeto\Teste unitario MGD\TU_MGD
Isso faz com q a hierarquia das pastas funcione corretamente na execução.

Obs. 7: A função de envio de notas ainda não está disponível. Temos apenas a visualização de notas. Para testar, insira notas diretamente no arquivo "dados\Notas.txt".

Obs. 8: O arquivo da análise do código está no diretório "Análise de Código" e deve ser aberto com o inFusion.

Obs. 9: Cada teste unitário já vem com os arquivos de dados mínimos necessários para sua execução.
